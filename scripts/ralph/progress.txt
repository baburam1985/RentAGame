## Codebase Patterns
- Game data lives in `web/src/data/games.ts` ‚Äî no API calls, pure static data
- Tests use vitest + @testing-library/react; `next/image` is mocked in `src/test/setup.ts`; `next/link` must be mocked per-test-file
- ESLint config was missing; added `web/eslint.config.js` using `eslint-config-next` flat config
- The layout wraps everything in `<CartProvider>` so all pages can access cart state
- GameDetailClient renders `GameDetails` twice (desktop sidebar + mobile stacked) ‚Äî use `getAllByText` in tests
- `aria-pressed` cannot be on elements with `role="listitem"`; use `<ul><li><button aria-pressed>` structure instead
- Use `useState(lazyInitFn)` for localStorage hydration ‚Äî calling `setItems` inside a `useEffect` triggers `react-hooks/set-state-in-effect` lint error
- ESLint `--max-warnings=0` is enforced; plain `<a href="/">` triggers `@next/next/no-html-link-for-pages` ‚Äî always use `<Link>` from next/link for internal routes
- For vi.mock calls in test files, the mock must be at the top level (not inside a function) due to hoisting; use a mutable module-level variable to control per-test mock state
